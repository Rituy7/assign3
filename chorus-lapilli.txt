For the Chorus Lapilli variant, I had to modify the original game to handle a lot of new rules. I started by adding logic to track how many pieces X and O had on the board at any time.

The logic is split into two main parts. First is the placement phase, where each player just puts down their first 3 pieces. After that, it switches to a movement phase. I had to implement a selected state to keep track of which piece a player wants to move. I also wrote an isAdjacent function using coordinate math to make sure pieces only move to an empty square that is right next to them (including diagonals).

The most complicated part was definitely the center square rule. I had to write a hasWinningMove helper to check if a player occupying the center square actually had a way to win the game on that turn. If they did, I added a check to make sure their move was either a winning move or a move that involved leaving the center square. It took a bit of testing to make sure the selection logic and the movement rules all worked together, but the game now handles all the specific Chorus Lapilli constraints.
